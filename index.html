Code Connect
  • PRODUCTIVITY
  • SOLUTION
  • TECH SPECS
  • MAKING OF

A large chunk of a developers' work time is spent reading code: We begin in a single file, where we read a function line by line. But then the program execution continues in another method, perhaps even in another file and another project. So we jump there and resume reading. And then we jump again, and again, until we understand how the program works.

Developers maintain a mental model of these relationships that exist within the source code.

The idea of Code Connect was born in my and my co-founder Josh's after having to quickly learn and understand the source code at each of the internships we did. Chris Granger's teaser video of Light Table had the feature of navigating on a function by function basis, but this feature never made it to the finished product. 

Code Connect is a code exploration tool that visualizes code on a function by function basis. It visualizes the call stack without running the program, allowing the developer to see functions called by a specified piece of code, as well as functions which lead the computer to execution of specified piece of code.

The function by function approach helps developers unravel complex or unfamiliar code. Developers use Code Connect to understand decompiled or obfuscated code. Developers use it to easier grasp the way software works.

Initially, we believed that Code Connect shines with large projects. When we built Code Connect and started using it, we discovered that it's useful for small projects as well - we used it to analyze the impact of changes committed by another developer.

DIY Ambilight
  • WHAT IS IT
  • MAKING OF
  • TECH SPECS

It is a clone of Philips Ambilight. Written a few years ago (= poorly coded) in C# and C++/CLI, built with a Teensy board, an LED strand and a sheet of acrylic cut to the shape of the screen. 

Making of.

The app takes screenshots and finds average color of each sector of the screen at a maximum rate of 19fps.

The screenshots are taken using Win32 API, which means that the project doesn't work with video games rendered using DirectX or OpenGL. DirectX screenshot module is high on absolutely slowly moving roadmap.
